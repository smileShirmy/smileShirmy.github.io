Wbepack Dev Server - 本地 rewrite规则

  • live reloading 触发浏览器自动刷新
  • 不能直接打包文件
  • dev-server是用于搭建本地的环境
  • 路径的重定向
  • 支持https
  • 浏览器中显示编译错误
  • 接口代理
  • 模块热更新 (live reloading是刷新浏览器，模块热更新不会刷新)

devServer

  • inline (默认为inline true模式)
  • contentBase (提供的内容的路径，如果是静态时要指定)
  • port (指端监听的端口)
  • historyApiFallback (指定rewrite规则)
  • https (指定为true，本地的证书，也可自己指定)
  • proxy (通过指定proxy进行远程端口来代理)
  • hot (模块热更新代码，就是把一些代码替换掉，一些框架是直接支持hot的，如果是自己写，需要去触发这个钩子)
  • openpage (指定webpack-dev-server最先打开的页面)
  • lazy (访问到的时候才打包，懒模式，多页面时非常有用，避免同时打包多个页面)
  • overlay (遮罩，给出编译错误的提示, ESLint检查时)

模块热更新

Module Hot Reloading

不刷新浏览器的前提下更新数据

  • 保持应用的数据状态
  • 节省调试时间

  • 样式调试更快

  • devServer.hot (置为true来使用)

  • webpack.HotModuleReplacementPlugin
  • webpack.NamedModulesPlugin (看到这些模块的相对路径)
  • module.hot (在代码中通过这个接口来对代码进行热更新的操作)
  • module.hot.accept(两个参数：依赖，依赖更新之后的回调)
  • module.hot.decline(拒绝给模块更新)

实际上 style-loader 已经包含了热更新，一些流行的框架也包括了热更新，除非使用原生的JS

可以在控制台看到一些模块热更新的提示

此时检查修改css样式，模块热更新还没有生效，因为没有使用style-loader，而是使用了extract，把它去掉。把loader恢复成最原始的样子

这时再修改样式文件就即时热更新了

hotOnly: true

设置 hotOnly 为 true，修改js文件不会进行热更行，需要加上热更新的代码

通过以上配置就不会有两个 url 了，实际上大部分都能让框架的loader帮忙完成，因此只要专注在业务代码上就可以了
