多页面应用配置

webpack 多页面应用

多页面应用的特点

  • 多入口 entry
  • 多页面 html
  • 每个页面不同的 chunk(引入的js和css不一样)
  • 每个页面不同的参数

实现多页面应用有两种方式

  1. 多配置
  2. 单配置(多个页面共享一份webpack配置)

多配置

  • webpack 3.1.0 开始支持
  • 可以处理一个配置的数组
  • 还可以借助工具来优化(parallel-webpack)

优点

  • 可以使用parallel-webpack来提高打包的速度
  • 配置更加独立，灵活 (单独配置所需要的配置)

缺点

  • 不能多页面之间共享代码，不能把公用代码提取出来

parallel-webpack

  • parallel-webpack –watch
  • parallel-webpack –config

单配置

优点

  • 可以共享各个entry之间的公用代码

缺点

  • 打包速度比较慢
  • 输出的内容比较复杂
