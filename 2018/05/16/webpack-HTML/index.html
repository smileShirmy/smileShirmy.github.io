生成HTML

  • 场景优化
  • 把提取的共用文件插入到HTML中（以后只打包业务代码）

HtmlWebpackPlugin

  • options:
  • template
  • filename(按照这个文件名来生成文件)
  • minify(是否压缩)
  • chunks(指定有哪几个chunk需要加到html中)
  • inject(是否把css、js通过标签的形式插到标签中)

安装：

  1. 运行webpack，此时输出的结果有一个index.html文件。
  2. 把文件名都修改为带有版本号的文件名，把inject设为false之后再进行打包，则看到成功把带有版本号的css和js注入到页面中了。
  3. 如果是多entry，如果不指定chunks，默认会把entry相关的chunks都载入到模板中。如果指定了入口，则只会把入口相关的文件插入到文件中。
  4. 打开网页发现css文件和js文件都不能正确定位，连接前面都多了一个dist目录。因为设定了pubilcPath为’dist/‘，因此所有的连接地址前面都多了一个dist/，此时刷新就没问题了。

配合优化
