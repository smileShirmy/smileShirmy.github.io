打包(提取)公共代码、代码分割和懒加载

打包(提取)公共代码

  • 减少代码冗余
  • 提高加载速度

避免重复加载公共代码

CommonsChunkPlugin

  • webpack.optimize.CommonsChunkPlugin

基本配置

  1. options.name or options.names (name/names 表示 chunk 的名称)
  2. options.filename (表示共用代码打包后的文件名)
  3. options.minChunks (出现的次数，具体的数值、Infinity、函数逻辑)
  4. options.chunks (指定提取代码的范围)
  5. options.children (子模块)
  6. options.deepChildren (所有模块)
  7. options.async (创建异步的公共代码流)

使用场景

  • 单页应用
  • 单页应用 + 第三方依赖
  • 多页应用 + 第三方依赖 + webpack生成代码

实际上对单个 entry 的打包是没有用的，单页面后面另有介绍

代码分割和懒加载

  • 十分重要的特性，尽量让用户加载更少的代码
  • 代码分割和懒加载是一回事
  • 并不是通过配置webpack，而是通过写代码的方式
  • webpack methods
  • ES 2015 Loader spec (动态 import)

webpack methods

  • require.ensure的四个参数：
  • []: dependencies
  • callback
  • errorCallback
  • chunkName
  • 对promise强依赖，如果浏览器不支持promise，需要加上polyfill
  • require.include
  • 分别加载子模块的时候先加载父模块，如果同时依赖一个模块，把这个模块移到父模块

ES 2015 Loader Spec

  • System.import() -> import() 现在更多是直接使用 import()
  • import() -> Promise 返回一个Promise
  • import().then() 可以使用then()使用动态的加载

  • 分离首次加载 和 访问后加载的代码
