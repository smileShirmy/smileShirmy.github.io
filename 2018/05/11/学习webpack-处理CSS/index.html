处理CSS style-loader

  • 引入
  • CSS modules
  • 配置less/sass
  • 提取CSS代码

之所以提取CSS代码而不直接import，因为文件会有自己的缓存，也是为了优化，那么用到这个样式的模块的时候，如果已经缓存了，那么就更快地加载。

CSS的代码也有可能是共用的，如果把CSS代码单独提出来，那么就不需要重复加载这部分CSS了。

处理CSS CSS-loader

options

  • alias (解析的别名)
  • importLoader (@import)
  • Minimize (是否压缩)
  • modules (是否启用css-modules)

CSS-Modules 语法

:local 定义本地(局部)的样式
:global 给定一个全局的样式
:compose 继承一段样式
:compose … from path 从某个文件中引入一些样式

minimize

压缩，这时样式没有空格了

处理CSS 提取 CSS

extract-loader

ExtractTextWebpackPlugin

第二种方式更常用

此时运行 webpack 可以看到两个文件

  • app.bundle.js
  • app.min.css

app.min.css 就是压缩之后并单独提取出来的 css 文件，但发现并没有把这个css文件加到文档中

此时在 index.html 文件中通过 标签引入

allChunks: false

注释掉 minimize

此时打包便可以看到三个文件：

  1. 0.bundle.js
  2. app.bundle.js
  3. app.min.js

发现文件名并没有改变

因此在 Output 中添加 chunkFilename 指定动态打包的文件名

此时得到 a.bundle.js

app.min.css 中没有 a.css 中的内容，这个内容跟 a.js 一起打包到 a.budnle.js，CSS写在JS中了

得到结论： 通过 allChunks 这个参数可以控制哪些CSS在一开始打包页面中，留一些动态加载的样式放在各个的模块中
