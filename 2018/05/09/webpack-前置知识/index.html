背景

Webpack的诞生

  • Tobias Koppers(github: @sokra)
  • GWT(Google Web Toolkit)
  • Pull request(没有人接受，所以自己开发一个)
  • Webpack

  • Instagram(发现了webpack，保养了它)

    It’s a funny story how I started with Webpack. Before getting addicted to JavaScript, I also developed in Java. I tried GWT (Google Web Toolkit) in that time. GWT is a Java-to-JavaScript Compiler, which has a great feature: code-splitting. I liked this feature and missed it in existing JavaScript tooling. I made a pull request to an existing module bundler, but it did not go through. Webpack was born.

前后端分工的变化

早年：

前端开发只需要产出什么？静态页面，数据动态套用逻辑都由后端工程师完成。前端只负责静态的页面和静态页面的动画效果例如轮播图，点击或是悬停动画，或者是不停向上滚动的文字等等。

现在：

前端开发越来越复杂，web 应用对前端的要求越来越高，前端开始除了静态页面的开发之外开始管理页面之间的逻辑，数据的状态，应用变的越来越复杂，前端逻辑开始抽象分层。
之前前端工程师只需要新建一个html 文件就可以开始编码，而现在前端需要搭建本地开发环境，接口代理，模块化开发。这些业务复杂度的提升导致前端开发方式的演变，从简单的页面构建到网站整理的架构规划，逻辑分层，模块化，从而需要构建工具的支持，来保证开发效率。

AMD/CMD/UMD

AMD Async module Definition：

  • 使用define定义模块
  • 使用require加载模块
  • RequireJS
  • 依赖前置，提前执行
  • https://github.com/amdjs/amdjs-api/wiki/AMD

Common Module Definition：

  • 一个文件为一个模块
  • 使用define来定义一个模块
  • 使用require来加载一个模块
  • SeaJS
  • 尽可能懒执行
  • https://github.com/cmdjs/specification/blob/master/draft/module.md

区别：

  • AMD提前执行了，CMD执行到的时候再执行
  • UMD Universal Module Definition
  • 通用解决方案

三个步骤：

  1. 判断是否支持AMD
  2. 判断是否支持 CommonJS
  3. 如果都没有，使用全局变量

Webpack的功能进化

webpack 核心概念

Entry

代码的入口、打包的入口、单个入口或多个入口(并非代表执行的入口)

依次找寻依赖

Output

  • 打包生成的文件(bundle)
  • 一个或多个
  • 自定义规则
  • 配合CDN

Loaders

  • 处理文件
  • 转化为模块

常用的 Loader

  • 编译相关
  • babel-loader、ts-loader
  • 样式相关
  • style-loader、css-loader、less-loader、postcss-loader
  • 文件相关
  • file-loader、url-loader

Plugins

  • 参与打包整个过程
  • 打包优化和压缩
  • 配置编译时的变量
  • 极其灵活

常用Plugins

  • 优化相关
  • CommonsChunkPlugin *(webpack4 SplitChunksPlugin)
  • UglifyjsWebpackPlugin
  • 功能相关
  • ExtractTextWebapckPlugin(打包成单独CSS文件)
  • HtmlWebpackPlugin(生成HTML)
  • HotModuleReplacementPlugin(热更新)
  • CopyWebpackPlugin(把第三方文件移到输出文件)

名词

  • Chunk：
  • 代码块，默认把代码分成一个个代码块
  • 如果用动态懒加载，懒加载的代码认为一个代码块
  • 相同的代码抽出来作为新的代码块
  • Bundle：
  • 打包过以后，一捆
  • Module：
  • 模块，通过loaders转为一个模块
