长缓存优化

什么是长缓存优化？

服务器返回请求的时候，告诉浏览器一些资源在一段时间内不需要更新它。长缓存优化的意义就是用户只下载那些需要更新的代码。

场景 一

希望改变app(业务逻辑)代码的时候不要去改变vendor(第三方)的代码

解决方法：

  • 提取vendor，hash -> chunkhash
  • 每次打包的时候在输出的界面上输出hash值，代表打包的哈希，而不是每个代码块的哈希，因此把哈希变成代码块的哈希
  • 提取 webpack runtime

总结

如果我们需要长缓存：

  • 独立打包 vendor
  • 抽出manifest(webpack runtime)因为修改代码webpack runtime可能会发生变化，因此要抽出来，可以把抽出来的manifest直接依赖到html里
  • 使用 NamedChunksPlugin
  • 使用 namedModulesPlugin
  • 对于动态模块，使用魔法注释给定模块名称

通过这两个plugin给顶我们的chunk和module不再使用webpack随机分配的id从0到几，保证chunkhash不会发生变化

这样就使打包的时候尽可能的保持长缓存，保证用户能尽可能地享受到这些缓存
