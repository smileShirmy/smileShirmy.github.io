Nginx是一个纯粹的Web服务器(HTTP服务)，编码更多是为了实现业务逻辑

Nginx代理配置和代理缓存的用处

代理缓存是在代理那层设置的，因此新的请求都会经过代理，如果已经缓存过一次了，在浏览器这边只要第一个用户请求在代理缓存中缓存过了，那么其它用户都可以使用代理缓存。

HTTPS解析

HTTP + security

HTTP本身的定义是明文传输，容易被篡改和截取

公钥：放在互联网上，所有人都可以拿到的一串加密的字符串，使用这个加密的字符串可以用来加密字符串

私钥：私钥只放在服务器上，只有有私钥才能解密到公钥加密的数据

实际的传输数据需要客户端和服务端通过主秘钥才能解密

HTTP2的优势和Nginx配置HTTP2的简单使用

优势：

  1. 信道复用
  2. 分帧传输(分成不同的帧，一起发送，根据帧信息进行组合，可以并发发送请求)
  3. Server Push(服务端主动)

HTTP1.1是串行发送的，因此非常慢

HTTP2没有一个阻塞的过程，并行发送
