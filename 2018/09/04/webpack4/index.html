代码切分

splitChunks

  • 新的chunk是被统统以来的，或是从node_modules中来的
  • 新的chunk大于30kb
  • 在按需加载的时候，最大的并行下载chunk数量应该要小于等于5
  • 在首屏加载的时候，最小的并行下载chunk数量应该要小于等于3

总结

  • Webpack4 可以零配置运行
  • Webpack4 打包速度比之前提高了90%
  • Webpack4 支持 Tree shaking ES6
  • Webpack4 RuntimeChunk / SplitChunks 更简单

未来

  • CSS 成为原生模块 (不再需要loader)
  • HTML 成为原生模块
  • URL/FILE 成为原生模块
  • 再也不需要 extract-text-webpack-plugin
  • 不需要 CSS FILE URL loader 了
  • 支持 CSS Code Splitting
  • HTML entry (指定页面作为入口)
